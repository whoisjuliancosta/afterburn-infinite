Julian's playtest feedback, six asks: (1) "add A/D steer" — with the nose mouse-owned, rotation would fight aim, so A/D = lateral strafe (flagged to Julian); (2) gems smaller; (3) legend screen must NOT be titled "PAYTABLE" ("that's stupid") — title it LEGEND; (4) a START button on the homescreen; (5) a place to enter your name for the leaderboard; (6) "every gem that enters that magnetic circle you eat" — gems inside the magnet field are a guaranteed catch.

Global constraints unchanged: pure-logic modules Node-clean; tuning in config.js; no per-frame shadowBlur (bake at init ok); playable with `assets/` missing; zero deps, no build step.

## T1. Controls + gem feel (config.js, input.js, ship.js, gems.js, main.js render)

1. **A/D strafe**: `SHIP.strafeMult: 0.75` in config (100% would make circle-strafing strictly dominant). input.js: `A`→`input.strafeLeft`, `D`→`input.strafeRight` (mirror the thrust/reverse key pattern incl. blur clearing). ship.js `updateShip`: accelerate perpendicular to the nose — left = nose angle − 90°, right = +90° — at `SHIP.thrust * ship.mods.engine * SHIP.strafeMult`. Boost does NOT scale strafe (forward-only afterburner, same as reverse). Both held → cancel. Tests: direction ⟂ nose (dot product ≈ 0 with nose vector), scaled by strafeMult and engine mod, off-flag no-op, L+R cancel.
2. **Smaller gems**: `GEMS.radius` 5 → 3.5 (pickup circle). Visual: main.js ~line 507 hardcodes `gemSize = GLOW.gemBlue.nat * 0.7` — replace with `GLOW.gemBlue.nat * 0.14 * GEMS.radius` (identical size at the old radius 5, shrinks with config — one knob). Check any other gem draw sites (legend pickup icons may stay fixed-size).
3. **Magnet lock — no escape** (gems.js `updateGems`, pure logic): when a gem is inside the effective magnet radius (the `magnetR` param already passed in):
   - it stops aging (`age` frozen or equivalent) — a gem inside the field can never blink out or expire;
   - closure is guaranteed: steer velocity toward the ship with speed at least `hypot(ship.vx, ship.vy) + GEMS.lockClosingSpeed` (new config const, ~260 px/s) — so even a boosting ship fleeing at max speed gets caught; cap acceleration feel with the existing magnetAccel ramp but enforce the floor speed once inside.
   - Once inside, always locked (leaving the radius because the ship dashed away doesn't unlock aging mid-frame — track `gem.locked = true` sticky flag).
   Tests: gem inside radius never expires over many seconds of dt; distance to a fleeing ship (moving at boost speed) strictly decreases once locked; gem outside radius ages normally; sticky lock survives radius exit; collection still fires on overlap.

## T2. Named leaderboard core (board.js, pure logic)

1. Board entries gain `name` (string). `recordRun(board, entry)` accepts `entry.name`, trims it, clamps to 12 chars, uppercases, falls back to `'PILOT'` when empty/missing. Sorting/placement (`placed`) logic unchanged.
2. **Legacy tolerance**: existing saved boards (localStorage) have entries without `name` — board.js display path must treat missing name as `'PILOT'`. No migration write needed.
3. Tests: name stored/trimmed/clamped/uppercased; fallback on empty/missing; legacy entries (no name) don't break recordRun/placed; sort order unaffected by names.

## T3. Menu UI — START button, name field, legend rename (hud.js, main.js)

1. **Legend title**: the legend screen header currently says "PAYTABLE" — retitle to **LEGEND** (keep the PAYS badges and the damage subtitle; only the screen name changes). Update any other user-facing "PAYTABLE" strings.
2. **START button**: `startRect(w, h)` in hud.js (single-source geometry like legendRect). Prominent accent-filled button labeled `START`, replacing the '— click to start —' text below the swatch row (LEGEND button moves below it or beside it — implementer arranges; nothing may overlap at 1280×800 or common sizes; keep everything legible). Click START or press **Enter** to start. **Click-anywhere-to-start is REMOVED** (the menu now has too many interactive elements: name field, hangar, LEGEND, START).
3. **Name field**: `nameRect(w, h)` in hud.js — a labeled input box (`PILOT NAME`) on the menu (sensible spot near the leaderboard/top area; implementer arranges). Behavior (main.js owns state):
   - Click the box to focus; focused box shows an accent border + blinking caret (drive blink off the existing clock, no Date).
   - While focused: printable A–Z / 0–9 / space / `-` append (uppercase, max 12 chars), Backspace deletes, Enter/Esc or clicking elsewhere blurs. **Keyboard shortcuts are suppressed while focused** — typing `L` must not open the legend, Enter-while-focused blurs rather than starting the game.
   - Persisted to localStorage key `np-shooter-name` (main.js composition root owns storage, same as best/board/paint). Loaded on boot; empty shows placeholder `PILOT` in dim text.
4. **Wire the name through**: on death, `recordRun` gets `name` from the menu field (fallback per T2). Menu top-3 lines become `1.  NAME  ·  1200  ·  wave 8`; the game-over board (drawGameOver) shows names the same way. Legacy no-name entries render as PILOT.
5. Controls text on the menu gains strafe: `W thrust · S reverse · A/D strafe · aim with mouse · hold SHIFT boost · right-click rocket` (keep it one legible line; split into two if it gets long).
6. Zero console errors with and without `assets/`; picker hit-rects unchanged; legend/L-key behavior unchanged except while the name field is focused.
