# V5.1 "Smooth & Deep" — Design Spec (perf pass + upgrade variety)

**Date:** 2026-07-10. Julian: FPS drops when the screen fills with bullets; wants broader upgrade variety. Plus two cosmetic minors from the v5 final review.

Global constraints unchanged. Visuals must be indistinguishable at rest — this is an optimization pass, not a restyle.

## A. Render performance

Root cause: per-frame `ctx.shadowBlur` on every bullet/enemy-shot/glow particle (Gaussian blur per draw call), 290 per-frame starfield arcs, per-entity save/restore.

1. **Pre-baked glow sprites.** At init (browser layer), render each glowing primitive ONCE into a small offscreen canvas WITH its glow (shadowBlur there is fine — it runs once): player bullet, enemy shot, boss shot, gem blue/red sparkle frames, glow particle dot (per color used: '#ffd75e', '#5fe8ff', '#ff9e3e', '#e6743e', '#8a879a', '#5ea8ff'). Runtime draws become plain `drawImage` — zero shadowBlur in the frame loop (grep-able acceptance: no `shadowBlur` assignment inside `render()` or anything it calls per-entity; thruster/rocket trail included).
2. **Starfield offscreen layer.** Pre-render each parallax layer's stars to an offscreen canvas at init; per frame, draw each layer canvas at its scrolled offset (wrapped, 2 draws per axis-wrap max). Twinkle: keep only the ~12 bright stars as individually drawn (cheap), give the static layers a subtle global alpha oscillation instead of per-star twinkle. Regenerate on resize.
3. **Batching.** Group draws by style: all bullets in one pass with one fillStyle/sprite, no save/restore per entity (symmetric sprites drawn without rotation transforms — bullets/shots/gems are symmetric; only ships/rockets rotate). Particles: one pass for non-glow (plain rects), one 'lighter' pass for glow — set composite twice per frame max, no per-particle state changes.
4. **Hard caps:** `FX.maxParticles` 320 (burst() drops oldest beyond cap — amend particles.js, pure logic + test), floaters already 40, enemy shots cap 220 (oldest culled — main), bullets uncapped (player-driven, bounded by fire rate cap).
5. **Measurement (T-verify):** headless driver drives a worst-case scene (?screen=stress dev hook: seeds wave 20-equivalent — ~40 enemies incl. a P3 boss, ~150 enemy shots, ricochet+spread+rapid at caps, ~200 bullets, 300 particles) and samples `requestAnimationFrame` deltas over 10s via page.evaluate; report p50/p95 frame time BEFORE (from the pre-branch commit, checked out into a temp worktree serve — or simpler: measure main-at-merge first, then after the perf commits) and AFTER on the same machine. Target: p95 ≤ 20ms in the stress scene; report honest numbers either way.

## B. Two v5 review minors

- Rocket right-click landing on a hit-pause frame is silently dropped: latch `rocketPending` in main until consumed (fire check already after pause return — keep placement, just don't lose the click).
- Range-end rocket detonation with zero hits: skip explosion FX/sfx (keep a small fizzle puff), so empty-air booms don't flash.

## C. Six new upgrades (pool 15 → 21)

| id | Name | Effect | Stacking |
|---|---|---|---|
| bigpayload | Big Payload | rocket `aoeRadius` ×1.4 | 2 stacks max (mods.rocketAoe) |
| fastreload | Fast Reload | rocket cooldown ×0.75 | 2 stacks max (mods.rocketReload, floor 2.2s) |
| burners | Efficient Burners | boost drain ×0.75 | 2 stacks max (mods.boostDrain) |
| lucky | Lucky Charm | gem drop chances ×1.3 (blue and red) | 2 stacks max (mods.luck) |
| rearguard | Rear Guard | auto-fire adds 1 bullet straight backward | 1 stack (mods.rear = 1) |
| adrenaline | Adrenaline | while below half HP: +15% fire rate and +10% max speed (dynamic, not clamped by CAPS since situational) | 1 stack (mods.adrenaline = 1) |

- Mods live on `createShip().mods` with neutral defaults (rocketAoe 1, rocketReload 1, boostDrain 1, luck 1, rear 0, adrenaline 0).
- Consumers: rockets.js reads effective aoe/cooldown via ship mods (fireRocket already receives ship; updateRockets needs the values captured at fire time or the store carries ship-derived values — pick, document, test). Boost drain: ship.js drain × mods.boostDrain. rollDrop gains a luck param (chances ×luck, clamped: blue ≤0.6, red ≤0.2). Rear Guard: fire() adds a bullet at angle+π on held fire only. Adrenaline: applied in updateShip/updateGun as dynamic multipliers when hp/maxHp < 0.5.
- rollOffers exclusions: each at its stack limit. Icons + card previews for all six (browser layer). Stat caps table (CAPS) untouched — new mods have their own stack limits.
